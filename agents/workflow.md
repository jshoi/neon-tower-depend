# Agent Workflow (오케스트레이터)

## 전체 파이프라인

```
사용자 요청
    │
    ▼
┌─────────────────────────────┐
│   시나리오 기획 에이전트      │  ← 메인 진행자
│   (scenario-agent.md)       │
└─────────────┬───────────────┘
              │ 레퍼런스 필요 시
              ▼
┌─────────────────────────────┐
│   벤치마킹 에이전트           │  ← 자료 수집
│   (benchmark-agent.md)      │
└─────────────┬───────────────┘
              │ 보고서 반환
              ▼
┌─────────────────────────────┐
│   시나리오 기획 에이전트      │  ← 기획서 작성
│   → feature-specs/ 저장     │
└─────────────┬───────────────┘
              │ 기획서 전달
              ▼
┌─────────────────────────────┐
│   기능 개발 에이전트          │  ← 구현 + 배포
│   (develop-agent.md)        │
└─────────────┬───────────────┘
              │ 완료 보고
              ▼
         사용자에게 결과 보고
```

---

## 실행 방법

### 방법 1: 전체 파이프라인 실행

사용자가 새 기능을 요청하면 아래 순서로 진행:

1. **시나리오 에이전트 실행** (Task tool, subagent_type: general)
   - `agents/scenario-agent.md` 지침 참조
   - `agents/game-state.md` 현재 상태 확인
   - 필요 시 벤치마킹 에이전트 호출

2. **벤치마킹 에이전트 실행** (Task tool, subagent_type: general)
   - `agents/benchmark-agent.md` 지침 참조
   - WebFetch로 레퍼런스 수집
   - 보고서를 시나리오 에이전트에 반환

3. **개발 에이전트 실행** (Task tool, subagent_type: general)
   - `agents/develop-agent.md` 지침 참조
   - 기획서 기반으로 구현
   - 빌드 + 배포

### 방법 2: 단계별 개별 실행

특정 단계만 실행하고 싶을 때:

```
"벤치마킹만 해줘: 보스 패턴 레퍼런스"
→ 벤치마킹 에이전트만 실행

"기획서 작성해줘: 보스 웨이브"
→ 시나리오 에이전트만 실행 (벤치마킹 결과가 있으면 활용)

"기획서 기반으로 개발해줘"
→ 개발 에이전트만 실행
```

## 폴더 구조

```
agents/
├── workflow.md            ← 이 파일 (오케스트레이터)
├── benchmark-agent.md     ← 벤치마킹 에이전트 지침
├── scenario-agent.md      ← 시나리오 기획 에이전트 지침
├── develop-agent.md       ← 기능 개발 에이전트 지침
├── game-state.md          ← 현재 게임 상태 스냅샷 (항상 최신 유지)
├── feature-specs/         ← 기획서 저장소
│   ├── boss-wave.md       ← 예시
│   ├── enemy-types.md     ← 예시
│   └── ...
└── benchmark-reports/     ← 벤치마킹 보고서 저장소
    ├── boss-patterns.md   ← 예시
    └── ...
```

## 에이전트 간 통신 형식

### 시나리오 → 벤치마킹 요청
```
[BENCHMARK REQUEST]
주제: <조사 주제>
키워드: <키워드1>, <키워드2>
목적: <왜 필요한지>
```

### 벤치마킹 → 시나리오 응답
```
[BENCHMARK REPORT]
주제: <주제>
카테고리: <분류>
## 핵심 발견 사항
...
## 시나리오 에이전트에 전달할 인사이트
...
```

### 시나리오 → 개발 전달
```
[TO: DEVELOP AGENT]
기획서 경로: agents/feature-specs/<기능명>.md
우선 구현 항목: <핵심 기능>
주의사항: <특이사항>
```

### 개발 → 사용자 완료 보고
```
[DEVELOP COMPLETE]
기능명: <구현 기능>
변경 파일: ...
배포 URL: https://jshoi.github.io/neon-tower-depend/
주요 변경사항: ...
```

## 우선순위 판단 기준

| 상황 | 액션 |
|------|------|
| 간단한 수치 변경 (밸런스 조정) | 개발 에이전트 직접 실행 |
| 알려진 기능 (백로그 있음) | 시나리오 에이전트 → 개발 에이전트 |
| 완전 새로운 기능 | 전체 파이프라인 (벤치마킹 포함) |
| 버그 수정 | 개발 에이전트 직접 실행 |

## 현재 구현 대기 중인 기능 (백로그)

우선순위에 따라 작업할 기능 목록. 완료 시 [완료] 표시.

### HIGH
- [ ] 보스 웨이브 (매 5웨이브)
- [ ] 다양한 적 유형 (빠른적, 탱커, 분열, 치유)
- [ ] 최고 기록 저장 (localStorage)

### MEDIUM  
- [ ] 사운드 효과 (Web Audio API)
- [ ] 특수 스킬 (액티브 쿨타임)
- [ ] 웨이브 예고 UI

### LOW
- [ ] 업적 시스템
- [ ] 스킨 시스템
- [ ] 리플레이 공유
